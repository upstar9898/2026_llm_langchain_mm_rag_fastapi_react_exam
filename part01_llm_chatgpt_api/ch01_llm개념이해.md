### 핵심 개념

"AI랑 ChatGPT랑 같은 건가요?" — 많이 받는 질문입니다. 이 네 가지는 **포함 관계**입니다. 가장 크고 넓은 개념부터 살펴봅시다.

```
AI (인공지능)
└── ML (머신러닝)
    └── DL (딥러닝)
        └── LLM (대규모 언어 모델)
                └── ChatGPT, Claude, Gemini ...
```

### 비유로 이해하기

|계층|비유|한 줄 설명|
|---|---|---|
|**AI**|"요리 분야 전체"|사람처럼 생각·판단하는 기계를 만드는 학문 전체|
|**ML**|"레시피 없이 먹어보고 배우는 요리사"|데이터를 보여주면 스스로 패턴을 학습하는 알고리즘|

|**LLM**|"책 수천억 권을 읽은 요리 로봇"|엄청난 양의 텍스트 데이터로 학습한 DL 언어 모델|

## 2. ChatGPT 작동 원리 — 토큰, 확률, 다음 단어 예측

### 2-1. 토큰(Token)이란?

ChatGPT는 텍스트를 **토큰** 단위로 처리합니다. 토큰은 단어와 비슷하지만, 정확히 같지는 않습니다.

- 영어: 보통 단어의 일부 (`play` → `play`, `ing` → 각각 1토큰)
- 한국어: 형태소 단위 (`학교에서` → `학교`, `에서` → 약 2토큰)
- 숫자: `123` → 보통 1~3토큰

**왜 중요한가?** — API 비용이 토큰 단위로 청구되기 때문입니다.

### 2-2. "다음 단어 예측" 기계

ChatGPT의 핵심 동작은 딱 하나입니다:

> **"지금까지 나온 단어들을 보고, 다음에 올 가장 자연스러운 단어를 예측한다"**

이걸 엄청난 텍스트 데이터로 수없이 반복 학습한 결과가 GPT입니다.

## 4. LLM의 한계 — 반드시 알고 시작해야 하는 것들

LLM을 실무에서 쓰려면 "못 하는 것"을 먼저 알아야 합니다. 이 한계를 극복하기 위해 Part 02~03에서 LangChain과 RAG를 배웁니다.

=======================================================
  LLM의 3가지 핵심 한계
=======================================================

[한계 1] 지식 컷오프 — 최신 정보를 모른다
---------------------------------------------
  질문    : 오늘 오전 2시 주차장 A구역 침입 사건 요약해줘
  LLM 학습 마감: 2024년 4월

  ❌ LLM 혼자서는 답 못함 — 오늘 CCTV 데이터가 없기 때문
  ✅ 해결책  : RAG (Part 03) — 우리 데이터를 실시간으로 제공

[한계 2] 환각 — 모르면 지어낸다
---------------------------------------------
  질문      : 2025년 3월 15일 서울 강남구 CCTV 침입 사건 번호 알려줘
  LLM 답변  : 2025-GN-0315-CV-00472 사건입니다.
  ⚠️  문제   : 존재하지 않는 번호를 그럴듯하게 만들어냈습니다

  ✅ 해결책  : 프롬프트에 '모르면 모른다고 해'를 명시 (CH03)
              + RAG로 실제 데이터 기반 답변 (Part 03)

[한계 3] 비용 — 모든 프레임에 LLM을 쓸 수 없다
---------------------------------------------
  가정: 1분마다 프레임 1장 추출 → 하루 1,440장
  프레임당 토큰 수: 약 150토큰

  하루 GPT-4o 비용: $0.54
  한달 GPT-4o 비용: $16.20

  ✅ 해결책  : OpenCV로 1차 필터링 → 이상 감지 프레임만 LLM 전달
              (이 강의의 핵심 구조!)

[한계 4] 컨텍스트 길이 — 한 번에 처리할 수 있는 양이 정해져 있다
---------------------------------------------
  GPT-4o 최대 컨텍스트: 128,000토큰
  탐지 결과 1건 ≈ 150토큰
  한 번에 처리 가능 최대 프레임: 약 853장

  ✅ 해결책  : LangChain 배치 처리로 나눠서 처리 (Part 02)
              + RAG로 필요한 것만 꺼내서 전달 (Part 03)
